# WisRight HRMS - POC UI/UX Design Specification

## Document Version: 1.0
**Date:** November 2025  
**Purpose:** Define UI screens, navigation, and user experience for POC

---

## 1. Design Principles

### 1.1 Core Principles
- **Clean & Minimal:** Focus on functionality over decoration
- **Intuitive Navigation:** Users should know where they are and where to go
- **Responsive:** Work on desktop (1920x1080) and tablet (768px+)
- **Accessible:** Clear labels, proper contrast, keyboard navigation
- **Consistent:** Reuse components, maintain visual hierarchy

### 1.2 Design System
- **UI Framework:** Material-UI (MUI) or Ant Design
- **Typography:** System fonts (Roboto for MUI, Inter for Ant Design)
- **Colors:** 
  - Primary: #1976d2 (blue)
  - Secondary: #dc004e (pink/red)
  - Success: #4caf50 (green)
  - Warning: #ff9800 (orange)
  - Error: #f44336 (red)
- **Spacing:** 8px base unit (8, 16, 24, 32, 48px)
- **Border Radius:** 4px for cards, 8px for buttons

---

## 2. Application Structure

### 2.1 Portal Types
1. **Admin Portal** (HR Admin, Super Admin)
2. **User Portal** (Employee, Manager)

### 2.2 Common Components
- Top Navigation Bar
- Side Navigation Menu
- Breadcrumbs
- Page Header with actions
- Data Tables with pagination
- Forms with validation
- Modal dialogs
- Toast notifications
- Loading states

## 6. Responsive Design

### 6.1 Breakpoints
- Desktop: 1200px+
- Tablet: 768px - 1199px
- Mobile: < 768px (out of scope for POC)

### 6.2 Tablet Adjustments
- Sidebar collapses to hamburger menu
- Table columns reduce (hide less important columns)
- Forms stack fields vertically
- Spacing reduces slightly

## 10. Accessibility Considerations

### 10.1 Basic Requirements
- Proper heading hierarchy (h1, h2, h3)
- Form labels associated with inputs
- Alt text for icons
- Keyboard navigation support
- Focus indicators visible
- Color contrast ratio > 4.5:1

### 10.2 ARIA Labels
- Use aria-label for icon buttons
- Use aria-describedby for form errors
- Use role="alert" for notifications
